# GitHub Copilot Instructions for heyChatApi Project

## 📋 Overview
This file defines guidelines and best practices for GitHub Copilot when making changes to the heyChatApi Spring Boot project. Follow these instructions to maintain code quality, consistency, and avoid unnecessary work.

---

## 🎯 Core Principles

### 1. **Only Do What Is Asked**
- ✅ Make ONLY the changes explicitly requested in the prompt
- ❌ Do NOT create extra documentation files (.md files)
- ❌ Do NOT commit code automatically
- ❌ Do NOT create PR descriptions or summaries
- ❌ Do NOT create test files unless specifically asked

### 2. **No Automatic Commits**
- Never run `git add` or `git commit` commands
- Never run `git push` commands
- User will manually commit when ready
- User will manually push when required

### 3. **No Documentation Generation**
- Do NOT create MIGRATION_SUMMARY.md files
- Do NOT create SETUP_GUIDE.md files
- Do NOT create any .md files unless explicitly requested
- Exception: Only create .md if prompt explicitly says "create documentation"

### 4. **Minimal Console Output**
- Avoid verbose explanations after code changes
- Keep responses brief and to the point
- Only explain what was changed, not what could be changed

## 📝 Logging Standards

### Logger Initialization (All Classes)
```java
private static final Logger logger = LoggerFactory.getLogger(ClassName.class);
```

### Logging Levels
- **`logger.info()`**: Important events (connection, subscription, message publish)
- **`logger.warn()`**: Warning situations (resource not found, validation issues)
- **`logger.error()`**: Error conditions (exceptions, failures)
- **`logger.debug()`**: Detailed debugging information (only when needed)

### Log Message Format
```java
// ✅ Good - Clear, informative
logger.info("Topic created - RoomId: {}, SessionId: {}", roomId, sessionId);

// ❌ Bad - Vague or unclear
logger.info("Topic stuff");
logger.info("Created");
```

### Section Logs
```java
// ✅ Good - Clear section separation
logger.info("=== WebSocket Message Received ===");
logger.info("Processing message...");

// ❌ Bad - No section separation
logger.info("Message received");
```

## 🎯 Java Coding Standards

### 1. **Naming Conventions**
```java
// Constants: UPPER_SNAKE_CASE
public static final String API_ENDPOINT = "/api";

// Classes: PascalCase
public class ChatController { }

// Methods: camelCase
public void sendMessage() { }

// Variables: camelCase
private String roomId;
private List<Message> messages;

// Package names: lowercase.reversed.domain
package hey.chat.config;
package hey.chat.controller;
```

### 2. **Variable Declaration**
```java
// ✅ Good - Private, final where applicable
private static final Logger logger = LoggerFactory.getLogger(Class.class);
private final RoomRepository roomRepository;
private String roomId;

// ❌ Bad - Public or non-final constants
public static String API_KEY;
static Logger logger;
```

### 3. **Method Organization**
```java
// Proper order in classes:
// 1. Constants
// 2. Logger
// 3. Autowired/Constructor injection
// 4. Constructor
// 5. Public methods
// 6. Private methods

public class ChatController {
    private static final Logger logger = LoggerFactory.getLogger(ChatController.class);
    
    private final MessageRepository messageRepository;
    private final RoomRepository roomRepository;
    
    public ChatController(MessageRepository msgRepo, RoomRepository roomRepo) {
        this.messageRepository = msgRepo;
        this.roomRepository = roomRepo;
    }
    
    @MessageMapping("/sendMessage/{roomId}")
    public Message sendMessage(...) { }
    
    private void validateRoom(String roomId) { }
}
```

### 4. **String Handling**
```java
// ✅ Good - Use constants for repeated strings
logger.info("Topic created: {}", Constants.TOPIC_ROOM_FORMAT, roomId);
logger.info("Room not found: {}", roomId);

// ❌ Bad - Magic strings everywhere
logger.info("Topic created");
throw new RuntimeException("room not found");
```

### 5. **Error Handling**
```java
// ✅ Good - Specific error handling
if(room.isPresent()) {
    // process
} else {
    logger.error("Room not found - RoomId: {}", roomId);
    throw new RuntimeException(Constants.MSG_ROOM_NOT_FOUND);
}

// ❌ Bad - Generic or no error logging
throw new RuntimeException("error");
```

### 6. **Null Checks**
```java
// ✅ Good - Proper null check
if (roomId != null && !roomId.trim().isEmpty()) {
    // process
}

// ✅ Better - Use Optional
Optional<Room> room = roomRepository.findByRoomId(roomId);
if (room.isPresent()) {
    // process
}

// ❌ Bad - No null check
roomId.equals("test"); // Could throw NullPointerException
```

### 7. **Comments**
```java
// ✅ Good - Explain WHY, not WHAT
// Using Optional instead of null check for NPE safety
Optional<Room> room = roomRepository.findByRoomId(roomId);

// ✅ Good - Document complex logic
// Generate unique session ID to track client connections
String sessionId = UUID.randomUUID().toString();

// ❌ Bad - Obvious comments
int count = 0; // Initialize count
String name = "John"; // Set name to John
```

## 📝 Exception Handling Standards

```java
// ✅ Good - Custom exceptions with context
public class RoomNotFoundException extends RuntimeException {
    public RoomNotFoundException(String roomId) {
        super("Room not found: " + roomId);
    }
}

// ✅ Good - Specific error handling with logging
try {
    Room room = roomRepository.findByRoomId(roomId)
        .orElseThrow(() -> new RoomNotFoundException(roomId));
} catch (RoomNotFoundException e) {
    logger.error("Room lookup failed: {}", e.getMessage());
    throw e;
}

// ❌ Bad - Generic exceptions
throw new Exception("Error");
throw new RuntimeException();
```

## 🎯 Database Standards

### Entity Definition
```java
// ✅ Good - Clear, documented entities
@Entity
@Table(name = "messages")
public class Message {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    @Column(name = "id")
    private Long id;
    
    @Column(name = "sender", nullable = false)
    private String sender;
    
    @Column(name = "content", length = 1000)
    private String content;
    
    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "room_id", nullable = false)
    private Room room;
}

// ❌ Bad - Missing annotations and constraints
public class Message {
    private Long id;
    private String sender;
    private String content;
    private Room room;
}
```

### Repository
```java
// ✅ Good - Clear method naming
@Repository
public interface MessageRepository extends JpaRepository<Message, Long> {
    List<Message> findByRoom_RoomId(String roomId);
    Optional<Message> findById(Long id);
}

// ❌ Bad - Vague naming
public interface MessageRepository extends JpaRepository<Message, Long> {
    List<Message> find(String roomId);
}
```
